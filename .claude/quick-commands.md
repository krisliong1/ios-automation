# 快速命令参考

> 使用这些简短命令来节省 token，快速执行常见操作

## 🎯 超简短命令格式

### 读取上下文
```
读配置
```
→ 读取 PROJECT_CONFIG.md，快速恢复项目状态

### 添加功能
```
新功能: [名称] - [简短描述]
位置: [路径]
```
→ 自动创建文件和基础代码

### 修改功能
```
改: [文件名] - [改什么]
```
→ 修改现有文件

### 提交推送
```
提交: [简短说明]
```
→ 自动 add、commit、push

### 创建文档
```
文档: [功能名]
```
→ 创建使用指南

## 📝 示例

### 示例 1: 添加新功能
```
新功能: 自动更新
位置: examples/Update/
功能: 检测并安装更新
```

### 示例 2: 修改现有代码
```
改: KrisAIFixer.swift
添加错误重试机制
```

### 示例 3: 快速提交
```
提交: 添加自动更新功能
```

## 🔧 常用缩写

- `读配置` = 读取项目配置
- `新功能` = 创建新功能
- `改` = 修改现有代码
- `提交` = git commit + push
- `文档` = 创建文档
- `测试` = 运行测试
- `查看` = 查看文件内容

## 💡 最佳实践

1. **开始对话时**:
   ```
   读配置 + [要做什么]
   ```

2. **添加功能时**:
   ```
   新功能: [名称]
   [1-2 行描述]
   ```

3. **结束任务时**:
   ```
   提交: [标题]
   ```

---

这样可以将原本需要几百个 token 的描述，压缩到几十个 token！
